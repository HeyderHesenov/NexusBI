# NexusBI çoxdilli dəstək (i18n) — dizayn

**Tarix:** 2026-07-01
**Status:** Təsdiqlənib (istifadəçi), icra: çox-agentli workflow.

## Kontekst / problem
Tətbiq tam Azərbaycan dilindədir (~325+ sabit UI mətni, 69 faylda; backend xəta/bildiriş
mesajları; AI-generasiya məzmunu AZ). İstifadəçi 3 yeni dil istəyir — **English, Русский,
Türkçe** — mövcud **Azərbaycan** ilə. Tələb: sağ-üstdə (tema toggle yanında) dil ikonu →
klikdə panel; **anında, problemsiz keçid**; struktur olduğu kimi qalsın; yüksək optimizasiya.

## Miqyas (təsdiqlənib)
**Hər şey:** tam frontend UI + backend istifadəçi mesajları + AI (YENİ generasiyalar seçilən
dildə). **Köhnə saxlanmış AZ məzmun (keçmiş insight/dashboard/qərar) tərcümə OLUNMUR** —
statik UI dərhal dəyişir, yeni AI cavabları seçilən dildə, köhnə məzmun olduğu kimi qalır.

## Memarlıq

### Frontend — react-i18next
- `i18next` + `react-i18next`. Dil resursları: `src/i18n/locales/{az,en,tr,ru}.json`.
- Açarlar **semantik + namespace**-li: `common.*`, `nav.*`, `topbar.*`, `query.*`,
  `dashboard.*`, `reports.*`, `search.*`, `pricing.*`, … (fayl/sahəyə görə qrup, toqquşma yox).
- `az` = **default + fallback** (mövcud mətnlər olduğu kimi az.json-a köçür).
- ~325+ sabit mətn → `t('açar')` / `<Trans>`. Struktur/komponent ağacı dəyişmir.
- **Optimizasiya:** yalnız aktiv dil chunk-ı lazy-load (dynamic import), fallback az əvvəldən;
  switch-də `i18n.changeLanguage` → re-render (reload YOX). Bundle-a yalnız kiçik i18next core.

### Dil seçici (UI)
- `TopBar`-da tema toggle-ın **solunda/yanında** `Globe`/`Languages` ikonu.
- Klikdə mövcud `components/ui/Dropdown.tsx` primitivi ilə panel: **AZ · English · Русский ·
  Türkçe**, aktivdə ✓. Klaviatura/a11y Dropdown-dan gəlir.
- `store/localeStore.ts` (zustand): `lang`, `setLang` → `i18n.changeLanguage` + `localStorage`
  (`nexusbi_lang`) + backend profilə (`PATCH /auth/me` və ya login-də göndərilir). No-flash
  inline init `index.html`/`main.tsx`-də.

### Backend
- `app/core/i18n.py`: `MESSAGES: dict[lang, dict[key, str]]` + `translate(key, lang)`.
- Locale mənbəyi: `X-Lang` header (client göndərir) → yoxdursa `user.language` → fallback `az`.
- `User.language` sütunu (+ migration, default `az`); `/auth/me` + update.
- `NexusBIException` mesajları tədricən kataloq açarları ilə (ən çox görünən istifadəçi mesajları).

### AI
- Generasiya prompt-larına dil təlimatı ("Respond in <language>") — `insight_generator`,
  `data_story`, `copilot`, `digest`, `root_cause`, `analysis` və s. Locale request-dən gəlir.
- YENİ cavablar seçilən dildə; köhnə saxlanmış AZ dəyişməz.

## Faz-faz icra
1. **Faz 1 — infra + switcher + tam frontend UI:** react-i18next quraşdır, `src/i18n/`,
   localeStore, TopBar switcher; 69 fayl string→`t()` + 4 dil JSON. Bulk **workflow** ilə
   (per-fayl agent: string çıxar→açar+AZ/EN/RU/TR qaytar; nəticələr JSON-a birləşir).
2. **Faz 2 — backend:** `User.language`+migration, `core/i18n.py` kataloq, locale resolve,
   mesajlar açarlara.
3. **Faz 3 — AI:** prompt dil inyeksiyası.
Hər faz: build/pytest/vitest + `code-review`(high) + commit + push.

## Optimizasiya prinsipləri
- Aktiv dil lazy-load (dynamic import), fallback az bundled.
- `changeLanguage` reload etmir; React re-render.
- Açar toqquşması yox (namespace); tərcümə faylları JSON (tree-shakable/lazy).

## Risklər
- ~325+ mətnin maşın-tərcüməsi (EN/RU/TR) — domen terminlərində kiçik keyfiyyət fərqləri
  ola bilər (sonra düzəldilir). JSX daxilində string→t() çevirmə JSX-i pozmamalıdır (agent
  təlimatı + build yoxlaması hər fayldan sonra).

## Verifikasiya
- `npm run build`, `npx vitest run`; hər dildə switch canlı (Playwright): TopBar/Sidebar +
  əsas səhifələr seçilən dildə, anında, 0 konsol xətası; backend locale mesajı; AI yeni cavab
  seçilən dildə. Faz sonu `code-review`(high).
